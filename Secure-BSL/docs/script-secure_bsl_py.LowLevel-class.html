Method Details [hide private]

comInit(self, port)

source code 

Tries to open the serial port given and initialises the port and variables. The timeout and the number of allowed errors is multiplied by 'aProlongFactor' after transmission of a command to give plenty of time to the micro controller to finish the command. Returns zero if the function is successful.

comDone(self)

source code 

Closes the used serial port. This function must be called at the end of a program, otherwise the serial port might not be released and can not be used in other programs. Returns zero if the function is successful.

comTxRx(self, cmd, dataOut, length)

source code 

Sends the command cmd with the data given in dataOut to the microcontroller and expects either an acknowledge or a frame with result from the microcontroller. The results are stored in dataIn (if not a NULL pointer is passed). In this routine all the necessary protocol stuff is handled. Returns zero if the function was successful.

bslSync(self, wait=0)

source code 

Transmits Synchronization character and expects to receive Acknowledge character if wait is 0 it must work the first time. otherwise if wait is 1 it is retried (forever).

bslTxRx(self, cmd, addr, length=0, blkout=None, wait=0)

source code 

Transmits a command (cmd) with its parameters: start-address (addr), length (len) and additional data (blkout) to boot loader. wait specified if the bsl sync should be tried once or repeated, forever Parameters return by boot loader are passed via blkin.
